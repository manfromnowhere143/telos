## The headline

**The value-wrong wall is not absolute.** Every earlier gold-free detector asked *"is this patch wrong?"* —
which requires predicting the correct value from the patch — and both scored `0/10` on the value subclass.

A patch-blind author asked *"what should this code do?"*, reading only the issue, caught one of the ten. The
test **passes on the accepted fix and fails on the certified-wrong patch**, on an input the graded suite never
covered. `iter226/django-11999`, by the OpenAI author.

So the ceiling is a property of gold-free detectors **that reason from the patch**, not of the value-wrong
class itself. The paper now says that; the correction was pre-committed in iter234's registration before any
data existed.

## Five things a successor must not misread

1. **`1/10`, not `3/10`.** The gold-free arm shows three value catches; **two of them are tests that fail on
   every implementation**, including gold. Their agreement with the label is coincidence. Always report the
   gold-validated arm as the finding and the gold-free arm as the deployable cost.
2. **The `0/54` gold-validated false-positive rate is not a virtue.** It is what remains after discarding
   every test that could not prove itself against a correct implementation — information a deployed system
   does not have. The deployable number is `13/54` union, *worse* than static judging's `5/54`.
3. **The independence endpoint is underpowered.** Pairwise Jaccard `0.0`/`0.0`/`0.5` and union-over-best of
   `+1` would suggest independent authors buy real coverage, but with one or two catches each that is
   indistinguishable from sparsity. **Do not cite it as an answer.** It needs more positives, not a better
   panel.
4. **iter234 is not TCP-1 and fills no iter212 gate.** Three models chosen by one agent, prompted by one
   prompt, in one workspace share a designer. The stop rule — *do not replace external humans with LLM
   judges* — stands untouched.
5. **The benchmark's gold-lookup shortcut** (iter233): `29` of `54` negatives are byte-identical to public
   gold. Compare detectors on the `25`-row hard subset, never the full negative set.

## Independence, decomposed — the useful part of the "what can a human do" question

Three axes, and they are not the same thing:

- **Information** independence (blind to patch/gold/labels/witnesses) — *achievable and verifiable*; iter234
  does it, and the guard proves it by parsing the prompt rather than trusting prose.
- **Architectural** independence (different model families) — *achievable*; iter234 uses three providers.
- **Institutional** independence (no shared incentive, attestable by a distinct party) — **irreducible.**

A fresh agent session in this workspace fails the first two: it can read `answers.json` and the witnesses, and
it is the same model family reasoning about the same artifacts. That is why it does not count as a second
author.

## What to pick up

- **A bigger positive set is now the bottleneck for everything.** `13` positives keeps every interval near
  `±0.2`, leaves the independence question unanswerable, and makes `1/10` an existence proof rather than a
  rate. More certified-yet-wrong patches would sharpen iter231–iter234 simultaneously. This is the highest-
  value next experiment and it costs solver runs.
- **Better issue-derived tests.** A third of them are simply wrong. A generate-validate-repair loop against
  the accepted fix would raise the gold-validated yield — but note that using gold to *repair* tests makes the
  instrument non-deployable, so it measures the mechanism's ceiling, not a system.
- **TCP-1** remains the endgame, now with its first direct evidence that the mechanism works. Operator-gated,
  needs human reviewers and budget, **not fakeable locally**.
